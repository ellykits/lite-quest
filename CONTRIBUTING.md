# Contributing to LiteQuest

Thank you for your interest in contributing to LiteQuest!

## Development Setup

1. Clone the repository
2. Open in IntelliJ IDEA or Android Studio
3. Sync Gradle dependencies
4. Run tests: `./gradlew test`

## Code Style

- Follow Kotlin coding conventions
- Use single imports (no wildcard imports)
- No unnecessary comments
- Keep code simple and maintainable

## Making Changes

1. Create a feature branch from `main`
2. Make your changes
3. Add tests for new functionality
4. Ensure all tests pass: `./gradlew test`
5. Update documentation if needed
6. Submit a pull request

## Pull Request Guidelines

- Provide a clear description of the changes
- Reference any related issues
- Ensure CI passes
- Keep PRs focused on a single feature/fix

## Testing

- Write tests for all new features
- Maintain test coverage
- Test on multiple platforms when relevant

## Documentation

- Update README.md for user-facing changes
- Add examples for new features
- Document public APIs

## Questions?

Open an issue for discussion before major changes.
